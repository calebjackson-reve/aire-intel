## 1. Error & Telemetry Surfaces

### ErrorLog (`prisma/schema.prisma`, model `ErrorLog`)
- **Fields:** `type` (api_failure | validation | sync | ui | ai | lofty | paragon | meta | dotloop | twilio | sendgrid | zapier), `source` (route/component), `message`, `context` (JSON: statusCode, requestBody, leadId), `stack`, `attempts`, `resolved`, `resolution`, `resolvedAt`, `createdAt`
- **Helper functions:** `logError()`, `withRetry()` (3 attempts, 500→1000→2000ms backoff), `detectPatterns()` (groups by `type::source`, severity if ≥2 in 24h), `getHealthScore()` (0-100 score + trend)
- **API surface:** `GET /api/errors?health=1` returns health score; system page polls every 30s
- **Loop use:** Oracle for error-rate-based loops; trigger for self-healing loops

### AgentRun (`prisma/schema.prisma`, model `AgentRun`)
- **Fields:** `agentType`, `status` (running|completed|failed|partial), `startedAt`, `completedAt`, `itemsProcessed`, `actionsQueued`, `errorLog` (JSON array), `durationMs`
- **API surface:** `GET /api/agents/runs` returns history
- **Loop use:** Trigger (on completion), oracle (status=failed → retry loop), memory (last run stats)

### ActionQueue (`prisma/schema.prisma`, model `ActionQueue`)
- **Fields:** `type` (draft_message|follow_up_text|send_client_email|post_content|create_lofty_task), `status` (pending|approved|executing|executed|skipped|failed), `requiresApproval`, `priority` (1-10), `briefDate`, `approvedAt`, `executedAt`, `skippedAt`, `failedAt`, `failReason`
- **Loop use:** Oracle (approval/skip ratio per action type), trigger (on items aging > threshold)
